# bench

Allocation benchmarks for luwes hot-path operations. Used to catch regressions
before they ship: the CI `bench` job enforces per-benchmark alloc ceilings.

## Benchmarks

### Raw luwes SDK

| Benchmark | allocs/op (fake) | allocs/op (CGO) | What it measures |
|-----------|-----------------|-----------------|-----------------|
| `BenchmarkHeaderAuthAccept` | 1 | 0 | Full accept path: filter pool, GetOneInto, Set |
| `BenchmarkHeaderAuthReject` | 2 | 2 | Reject path: SendLocalResponse body + header slice |
| `BenchmarkGetOne` | 0 | 1 | `HeaderMap.GetOne` — allocates on CGO (valueView escapes) |
| `BenchmarkGetOneInto` | 0 | 0 | `HeaderMap.GetOneInto` — zero-alloc even on CGO |
| `BenchmarkGet` | 1 | 1 | `HeaderMap.Get` — always allocates (multi-value slice) |
| `BenchmarkGetMiss` | 0 | 0 | `HeaderMap.Get` miss — returns nil, no alloc |
| `BenchmarkGetAll` | 1 | 1 | `HeaderMap.GetAll` — one alloc for output slice |
| `BenchmarkGetChunks` | 0 | 0 | `BodyBuffer.GetChunks` — stack scratch, no alloc |
| `BenchmarkFilterCreate` | 1 | 0 | `Factory.Create` — pool miss (GC cleared); steady-state: 0 |

### sahl ergonomic layer

| Benchmark | allocs/op (fake) | allocs/op (CGO) | What it measures |
|-----------|-----------------|-----------------|-----------------|
| `BenchmarkSahlHandlerNoOp` | 4 | 3 | Baseline: Method+Path+Host pre-copies |
| `BenchmarkSahlHandlerAccept` | 5 | 3 | Accept via Peek: NoOp + 1 fake buf escape |
| `BenchmarkSahlHandlerReject` | 6 | 4 | Reject via Send: +1 SendLocalResponse body |
| `BenchmarkSahlHeaderPeek` | 5 | 3 | Peek contribution: same as Accept (0 extra on CGO) |
| `BenchmarkSahlHeaderGet` | 6 | 4 | First Get: +1 ToString into cache |
| `BenchmarkSahlHeaderGetCached` | 6 | 4 | Cached Get: 0 additional allocs (cache hit) |

**Fake vs CGO discrepancy:** on the fake (pure Go) path, `var buf UnsafeEnvoyBuffer`
escapes to heap because `GetOneInto` is dispatched through a `HeaderMap` interface,
blocking inlining. On the real CGO path the call is direct and `buf` stays on the
goroutine stack. The CGO numbers are the production numbers.

## Run

```sh
# Run all benchmarks, 5 iterations, save results:
make bench

# Single benchmark:
go test -bench=BenchmarkHeaderAuthAccept -benchmem -count=5 ./bench/

# With memory profiling:
go test -bench=BenchmarkHeaderAuthAccept -benchmem -memprofile=/tmp/allocs.out ./bench/
go tool pprof -alloc_objects -http=:8080 /tmp/allocs.out
```

## Alloc regression gate (CI)

The `bench` CI job enforces per-benchmark ceilings. Any benchmark exceeding
its ceiling fails the build. Current ceilings are in `.github/workflows/ci.yml`
under the `alloc regression gate` step.

## File structure

```
bench/
  bench_test.go       raw luwes SDK benchmarks (BenchmarkHeaderAuth*, BenchmarkGet*)
  sahl_bench_test.go  sahl layer benchmarks (BenchmarkSahl*)
  profiles/           flamegraph SVGs (generated by make flamegraph)
  results.txt         latest bench output (updated by make bench, gitignored)
```
